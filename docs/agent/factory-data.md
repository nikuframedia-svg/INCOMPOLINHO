## 4. Constraints — Descrição Detalhada

### Constraint 1: SetupCrew (HARD)
- **Capacidade**: 1 setup simultâneo em TODA a fábrica
- **Recurso**: Virtual SETUPCREW, partilhado por todas as máquinas
- **Lógica**: Quando uma máquina está a fazer setup (troca de ferramenta), nenhuma outra pode iniciar setup
- **Resolução**: Tentar turno alternativo, depois dia seguinte. Se impossível: INFEASIBLE

### Constraint 2: ToolTimeline (HARD)
- **Regra**: A mesma ferramenta não pode estar em 2 máquinas simultaneamente
- **Exemplo**: BFP079 está em PRM031. Se PRM039 (alt) precisa de BFP079, tem de esperar que PRM031 termine
- **Resolução**: Adiar início até a ferramenta estar livre

### Constraint 3: CalcoTimeline (HARD)
- **Regra**: O mesmo calço não pode ser usado em 2 máquinas ao mesmo tempo
- **Lógica**: Calços são moldes partilhados entre ferramentas
- **Resolução**: Adiar início até o calço estar livre

### Constraint 4: OperatorPool (ADVISORY)
- **Regra**: Soma de operadores por turno não excede pool da área (PG1/PG2)
- **Modo**: Advisory — regista excesso mas NÃO bloqueia scheduling
- **Impacto**: 14 ferramentas requerem 2 operadores (VUL038, VUL111, VUL115, VUL127, BFP188, BFP195, EBR001, JDE002, LEC002)

## 5. Exemplo Completo: Interpretação de um SKU do ISOP

### SKU: 1064169X100 (Front Link HA With Bushings LH)

**Dados Mestre (colunas A-M)**:
- Cliente: FAURECIA (210020)
- Máquina: PRM031 (Prensa Grande, área PG2)
- Ferramenta: BFP079
- Cadência: 1681 peças/hora
- Operadores: 1
- Lote económico: 36400
- Peça gémea: 1064186X100 (Front Link RH)
- Alt. máquina: PRM039

**Valores NP nas datas (colunas O+)**:
```
Dia:  27/02  28/02  01/03  02/03  03/03  04/03  05/03  06/03  ...  10/03  ...  13/03
NP:   2751   2751   2751   2751   2751   2751  -15600 -15600  ... -10400  ... -18200
```

**Leitura correcta**:
1. **Dias 27/02 a 04/03**: NP = +2751 → stock de 2751 peças. Sem encomenda.
2. **Dia 05/03**: NP = -15600 → ENCOMENDA de 15600 peças, deadline 05/03
   - Taxa efectiva = 1681 × 0.66 = 1109 pcs/hora
   - Tempo = 15600 / 1109 = 14.1 horas
   - Com DAY_CAP = 17h, precisa de ~1 dia
3. **Dia 06/03**: NP = -15600 → OUTRA ENCOMENDA de 15600 peças (repetido CONTA)
4. **Dia 10/03**: NP = -10400 → NOVA encomenda de 10400 peças
5. **Dia 13/03**: NP = -18200 → NOVA encomenda de 18200 peças

**Scheduling com gémea**:
- Gémea: 1064186X100 (mesma tool BFP079, mesma máquina PRM031)
- Emparelhamento cross-EDD: 1ª encomenda de A com 1ª de B, 2ª com 2ª, etc.
- Cada corrida: tempo = max(qty_A, qty_B) / (1681 × 0.66)
- Output separado: A recebe |NP_A|, B recebe |NP_B|

**O que NUNCA fazer**:
- Somar todos os negativos (-15600 + -15600 + -10400 + -18200 = -59800) como "demand total"
- Calcular delta entre +2751 e -15600 (= 18351) como quantidade a produzir
- Usar lote económico (36400) para inflacionar produção
- Apresentar tudo como um bloco único de 578h no Gantt
